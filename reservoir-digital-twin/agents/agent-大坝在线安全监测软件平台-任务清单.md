# 大坝在线安全监测软件平台 - 任务清单 v0.3

说明：基于需求草案，分解为可执行的子任务，明确阶段产出与负责人，便于后续 CI/CD 验证。

## 1) 需求确认与范围界定
- 明确 MVP 边界、非功能性需求（性能、安全、可维护性）及验收标准。
- 确认数据源优先级、关键监测点及核心告警场景。

## 2) 架构设计与技术选型
- 以微服务/后端分层架构为基础，确定数据库、消息队列、API 网关、鉴权方案。
- 选型初稿：PostgreSQL + PostGIS、TimescaleDB、CesiumJS、Redis、OpenAPI、JWT/RBAC。

## 3) 数据字典草案落地
- 将字段、表结构、关系以数据字典形式落地，与开发对齐。
- 产出：数据字典初稿文档（agent-大坝在线安全监测软件平台-数据字典初稿.md）

## 4) API 合同草案
- 定义核心资源及 CURD、列表、过滤、分页等契约。
- 产出：API 草案文档（可供后续实现对照）

## 5) MVP 开发任务分解

### 后端开发任务
- 5.1 数据模型搭建（Dam、MonitoringPoint、Sensor、Timeseries、Alarm 等）
- 5.2 数据接入管道与时序数据写入（Timeseries ingestion）
- 5.3 基础 API：/dams、/monitoring-points、/timeseries、/alarms 等
- 5.4 三维场景占位与前端骨架（Cesium/Three.js 结合）
- 5.5 用户鉴权：JWT/OAuth2 初版实现
- 5.6 基础监控告警与面板
- 5.7 报表模板与示例导出
- 5.8 CI/CD 和本地部署脚本初稿

### 前端开发任务
- 5.9 基础框架搭建
  - 5.9.1 Next.js 14 项目初始化与配置
  - 5.9.2 设计系统样式（CSS Variables、主题切换）
  - 5.9.3 App Shell 布局（左侧导航 + 顶部栏）
  
- 5.10 核心页面开发
  - 5.10.1 首页 / 导航入口
  - 5.10.2 水库大坝列表页 /dams
  - 5.10.3 水库详情页 /dams/[id]
  - 5.10.4 数据看板 /dashboard
  - 5.10.5 告警中心 /alerts
  - 5.10.6 预测分析 /forecasts
  - 5.10.7 系统管理 /admin

- 5.11 组件库建设
  - 5.11.1 通用卡片组件 Card
  - 5.11.2 统计卡片组件 StatCard
  - 5.11.3 表格组件 Table
  - 5.11.4 图表组件 TimeSeriesChart
  - 5.11.5 地图/3D 占位组件
  - 5.11.6 主题切换组件 ThemeToggle

- 5.12 API 对接
  - 5.12.1 前端 API 封装（使用 SWR）
  - 5.12.2 Mock 数据与接口契约对齐
  - 5.12.3 错误处理与加载状态

- 5.13 主题与国际化
  - 5.13.1 Light/Dark 主题切换
  - 5.13.2 中文界面优化

### API 合同对齐
- 5.14 前端对接 /api/dams 的 mock 数据，确保契约一致

## 6) 验证与测试用例设计
- 为各核心模块编写单元测试、集成测试与端到端测试用例框架。
- 覆盖数据正确性、接口稳定性、告警正确性、以及 3D 渲染性能的基线测试。

## 7) 文档与上线准备
- 使用文档、部署文档、API 文档、用户手册等整理。
- 准备演示用例、验收材料与上线回滚方案。

## 8) 里程碑与交付
- MVP1: 核心数据模型、基础 API、3D 场景占位、鉴权、数据接入占位、前端基础框架
- MVP2: 3D 实时绑定、告警面板、巡查路线、报表样例、完整前端页面
- MVP3: 预测/会商/预案、跨系统工作流、完整测试集合
